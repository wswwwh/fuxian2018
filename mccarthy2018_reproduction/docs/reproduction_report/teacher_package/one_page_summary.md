# 一页总结

本项目的目标是复现 McCarthy 2018 关于日地、地月系统拟周期轨道的论文图谱，并把工作从“生成相似图片”推进到“能追溯每张图的数据来源、残差证据、Jacobi 一致性、周期性或稳定性证据以及 proxy 边界”。当前已经完成 54 张目标图的一版工程化覆盖，建立了逐图验证表 `data/computed/figure_validation_table.csv`，并形成主报告、逐图状态附录、数值审计附录、proxy 使用附录、未来计划、组会大纲和问答稿。这个成果可以作为阶段性复现报告包和教师展示包使用，但不能表述为 complete McCarthy 2018 numerical reproduction。

54 图的当前分类体现了项目的证据强弱。16 张图属于 numerical reproduction，主要覆盖 CR3BP 基础、平衡点、零速度曲线、部分周期轨道族和若干 Chapter 3 corrected numerical families。6 张图属于 physical-consistency baseline，说明物理机制和局部数值链条可信，但尚未证明与 McCarthy 原始数值完全一致。Fig. 3.16 和 Fig. 3.17 单独标为 physical-consistency baseline (partial)，因为高幅值 quasi-DRO 分支没有通过审计。另有 17 张图是 shape-match with local numerical overlay，13 张图是 proxy/schematic only。这个分级的主要作用，是防止把视觉相似度误当成完整数值复现。

三个审计案例最适合向老师说明项目价值。第一，Fig. 3.10 已建立 q=2、q=3、q=8 period-q halo 的局部数值近似。q=2 和 q=3 的 residual、Jacobi drift 和 closure 较好；但 q=8 虽然多段打靶 patch residual 接近 roundoff，单弧全周期闭合误差仍为 `3.906984451743337`，因此只能称为 highly unstable multiple-shooting local approximation，不能称为稳健周期轨道。第二，Chapter 4 DG manifold 已记录 source residual、DG mapping、unstable multiplier、growth ratio 和 Jacobi drift，支持 physical-consistency baseline；但图中仍保留 proxy background 或 grey reference，thesis-scale dense global manifold sheet 尚未完成。第三，Chapter 3 quasi-DRO 是当前最主要瓶颈。accepted branch 只到 `max_abs_z = 10164.02309965055 km`，没有 accepted member 超过 10,500 km 或 11,000 km。`N=61` 能降低 Fourier tail energy，但不能推进接受分支；fixed-rotation 高幅值候选能到 11,000 km 以上，却不能通过 residual / Jacobi audit。当前较可信解释是 fixed-mapping-time formulation 在 rho 约 `1.44388` 附近遇到参数化或 Newton basin failure。

Route A 原始数据搜索已经检查 Purdue Hammer/Figshare thesis record、相关论文、JPL periodic-orbits API 和 GitHub 等来源，没有找到 McCarthy 原始 quasi-DRO branch data、initial states、appendix tables 或 official code。Fig. 3.17 因此只被 digitize 为 lower-authority reference trend，用来记录论文图趋势与 accepted corrected branch 之间的差距；它不是 raw branch data。Fig. 3.16 是静态三维图，缺少相机、投影和原始分支信息，不能精确 digitize。下一步建议先继续有边界的 Route A 搜索和负面证据记录；若仍无原始数据，再把 Route B formulation replacement 作为独立算法任务，先写清 residual equations、continuation variables、phase constraints 和 acceptance thresholds，再尝试突破 quasi-DRO 10,500 / 11,000 km bottleneck。

汇报时建议采用克制但清楚的叙述：本项目的主要贡献不是宣称已经完全复现论文，而是把论文图、代码模块、计算数据、诊断图和报告结论组织成可检查的证据链。最适合展示的成果是 54 图分类表、三个数值审计案例、Fig. 3.17 digitized comparison、Route A/B/C 路线图和 proxy usage summary。需要反复说明的边界包括：展示图副本不是新的数据源，digitized trend 只作为参考曲线，Chapter 5 仍是 baseline/proxy 应用层，后续算法突破必须建立在明确的审计门槛之上。这样汇报既能体现工程化复现进度，也能避免把尚未通过审计的高幅值分支包装成确定结论。若老师希望继续推进，优先问题应是原始数据缺口和 quasi-DRO formulation，而不是继续美化图像；所有新结论都应先回到验证表和审计附录中闭环。
